================================================================================
KEY FEATURES
================================================================================

1. TICKETING SYSTEM
   - Create and assign tasks
   - Manage queries
   - Track status (open, in progress, resolved, closed)
   - Priority levels (low, medium, high, urgent)

2. PATIENT ASSIGNMENT
   - Assign patients to doctors
   - Bulk assignment
   - Track doctor workload
   - View unassigned patients

3. DEPARTMENT & TEAM MANAGEMENT
   - Create departments
   - Assign department heads
   - Create specialized teams (Clinical, Emergency, Surgical, Pediatric, ICU, etc.)
   - Assign team leads and members

4. AI CLINICAL ASSISTANT (Doctors only)
   - Patient history summary
   - Lab results analysis
   - Treatment recommendations
   - Risk factor assessment
   - Clinical question answering

5. DOCTOR NOTES
   - Clinical notes on patients
   - Note types: consultation, diagnosis, treatment, follow-up, observation
   - Private notes option

6. HEALTH CHATBOT (Dr. Tega)
   - Available to ALL users
   - General health questions
   - Health tips (nutrition, exercise, mental health, sleep, preventive care)
   - Medical condition explanations
   - 24/7 availability

7. PERSONAL DIARY
   - Available to ALL users
   - Text entries
   - Audio recordings
   - Video recordings
   - Mood tracking
   - Tags and organization
   - Completely private

================================================================================
TECHNICAL INFORMATION
================================================================================

Frontend: React + TypeScript + Vite
Backend: Python FastAPI
Database: SQLite (local_dev.db)
AI: OpenAI GPT-4o-mini

Frontend Port: 5174
Backend Port: 5000

================================================================================
NOTES
================================================================================

- All passwords are stored in plain text for demo purposes
- In production, passwords should be hashed
- Media files (audio/video) are stored locally in media/diary folder
- OpenAI API key is configured in .env file
- Database can be reset by deleting local_dev.db and restarting backend
